# USER.md - 사용자 정보 / 선호

- **호칭:** 대표님
- **요구 톤:** 항상 존댓말, 친근한 어투, 짧고 명확한 문장 선호
- **프로젝트 의도:** OpenClaw(김빌드 과장)는 메신저/감독관, 실제 개발은 OpenCode가 스킬 워크플로(exec + process + JSON-RPC)로 수행
- **개발 방식 선호:** 여러 모델/역할을 섞는 에이전트 오케스트레이션 + 스펙 기반 개발
- **중요 원칙:** 보안(포트 외부 오픈 금지, 비밀 평문 저장 금지), 운영 안정성(로그/백업/복구) 우선

## 텔레그램에서 OpenCode 사용법 (스킬 방식)

**/acp spawn, /focus 없음.** "계획해줘", "구현해줘" 등 **자연어만** 보내면 김빌드가 OPENCODE_ACP_WORKFLOW.md에 따라 OpenCode를 띄우고, 결과를 요약해 전달합니다. 같은 채팅에서 이어서 지시하면 됩니다.

- 예: "기존 아케이드게임센터 코드 구현 상태 확인하고 완성 계획 수립해줘"
- 예: "테스트로 계획만 세워줘"

상세 절차는 워크스페이스 OPENCODE_ACP_WORKFLOW.md 참고.
